1. fork()
   → child created (copy of shell)

2. child enters if(pid == 0)

3. execvp("ls", ...)
   → KERNEL loads /bin/ls into THIS process

4. your shell code in child = GONE

5. now CPU starts executing ls's main()

6. ls prints output

7. ls calls exit()

8. child process dies

9. parent waitpid() returns
